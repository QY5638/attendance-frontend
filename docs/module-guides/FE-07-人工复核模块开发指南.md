# FE-07 人工复核模块开发指南

## 1. 文档说明

- 本文件用于指导 `FE-07` 的独立开发。
- 开发前先读取前端并发清单、后端复核接口文档和异常详情契约。

## 2. 模块目标

- 实现复核详情、AI 辅助建议展示、复核提交和复核历史展示。

## 3. 模块边界

- 只做复核处理页和反馈学习入口。
- 不承载异常列表页。

## 4. 前置依赖

- `FE-01`
- 后端依赖 `BE-07`

## 5. 固定接口清单

1. `POST /api/review/submit`
2. `GET /api/review/{exceptionId}`
3. `GET /api/review/{exceptionId}/assistant`
4. `POST /api/review/feedback`
5. `GET /api/exception/{id}`

## 6. 固定字段与状态约定

- 页面必须基于 `exceptionId` 打开。
- 表单核心字段为 `reviewResult`、`reviewComment`。
- 页面需展示 `aiReviewSuggestion`、`similarCaseSummary` 和反馈标签入口。

## 7. 代码目录范围

- `src/views/review/`
- `src/api/`
- `src/components/`

## 8. 详细开发步骤

1. 搭建复核详情布局。
2. 接入异常详情和复核历史查询。
3. 接入 AI 复核辅助接口。
4. 实现确认、驳回表单。
5. 接入复核提交接口。
6. 增加反馈标签和策略反馈提交入口。
7. 处理提交成功、失败和重复提交场景。

## 9. 最小验证命令

```bash
npm run build
```

## 10. 完成标准

- 复核流程页面闭环。
- AI 复核建议可见。
- 提交结果与后端联动正确。
- 反馈标签可提交。

## 11. 禁止改动项

- 不在本模块重复实现异常列表页。

## 12. 文档同步要求

- `exceptionId`、复核字段和状态文案必须与 API 文档一致。
