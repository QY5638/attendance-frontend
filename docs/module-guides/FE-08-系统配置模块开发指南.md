# FE-08 系统配置模块开发指南

## 1. 文档说明

- 本文件用于指导 `FE-08` 的独立开发。
- 开发前先读取前端并发清单、后端系统配置相关接口文档。

## 2. 模块目标

- 完成 `/system` 下的设备管理、规则配置、提示词模板、风险等级、异常类型和日志查询页面。

## 3. 模块边界

- 只做系统配置区页面。
- 不侵入用户管理、打卡、异常主业务页。
- 承载模型策略和模型日志管理入口。

## 4. 前置依赖

- `FE-01`
- 后端依赖 `BE-02`、`BE-05`、`BE-06`、`BE-07`、`BE-08`

## 5. 固定接口清单

- 设备接口组。
- 规则接口组。
- `GET /api/system/prompt/list`
- `POST /api/system/prompt/add`
- `PUT /api/system/prompt/update`
- `PUT /api/system/prompt/status`
- 风险等级接口组。
- 异常类型接口组。
- `GET /api/log/operation/list`
- `GET /api/system/model-log/list`

## 6. 固定字段与状态约定

- 设备页面统一使用 `deviceId`。
- 日志接口由后端统计与审计能力提供。
- 模型页需展示模板场景、版本、状态和更新时间。
- 系统页需按子页组织，不做单页混杂实现。

## 7. 代码目录范围

- `src/views/system/`
- `src/api/`
- `src/components/`

## 8. 详细开发步骤

1. 搭建 `/system` 页面结构和导航。
2. 实现设备管理子页。
3. 实现规则配置子页。
4. 实现提示词模板管理子页。
5. 实现风险等级配置子页。
6. 实现异常类型配置子页。
7. 实现模型调用日志子页。
8. 实现操作日志查询子页。
9. 统一系统页提示、权限显示和交互反馈。

## 9. 最小验证命令

```bash
npm run build
```

## 10. 完成标准

- `/system` 下子页齐全。
- 查询和更新形成页面闭环。
- 提示词模板和模型日志可展示。
- 日志页可直接联调后端。

## 11. 禁止改动项

- 不把用户管理页混入系统配置页。
- 不在本模块实现统计大屏。
- 不在本模块重做异常详情和复核页面。

## 12. 文档同步要求

- 若设备、规则、风险等级、异常类型、日志字段变化，需先对齐后端 API 文档。
