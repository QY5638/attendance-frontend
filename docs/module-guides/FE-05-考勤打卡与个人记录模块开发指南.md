# FE-05 考勤打卡与个人记录模块开发指南

## 1. 文档说明

- 本文件用于指导 `FE-05` 的独立开发。
- 开发前先读取前端并发清单、后端打卡与补卡接口文档。

## 2. 模块目标

- 实现打卡、个人记录查看和补卡申请。

## 3. 模块边界

- 只做打卡提交和个人记录。
- 不做异常、预警、复核页面。

## 4. 前置依赖

- `FE-01`
- 后端依赖 `BE-04`

## 5. 固定接口清单

1. `POST /api/attendance/checkin`
2. `GET /api/attendance/record/{userId}`
3. `GET /api/attendance/list`
4. `POST /api/attendance/repair`
5. `POST /api/face/verify`

## 6. 固定字段与状态约定

- 打卡请求必须传递 `deviceId`。
- 个人记录页需展示时间、类型、状态。
- 补卡表单字段以 API 文档为准。

## 7. 代码目录范围

- `src/views/attendance/`
- `src/api/`
- `src/components/`

## 8. 详细开发步骤

1. 搭建打卡页和个人记录页。
2. 接入打卡提交接口。
3. 接入个人记录查询。
4. 实现补卡弹窗和表单。
5. 处理成功、失败、重复提交和无记录场景。

## 9. 最小验证命令

```bash
npm run build
```

## 10. 完成标准

- 打卡可提交。
- 个人记录可查看。
- 补卡可申请。

## 11. 禁止改动项

- 不在本模块实现异常列表和预警页面。

## 12. 文档同步要求

- 发现补卡或记录字段差异时，先对齐后端 API 文档再调整页面。
