# FE-03 用户部门角色模块开发指南

## 1. 文档说明

- 本文件用于指导 `FE-03` 的独立开发。
- 开发前先读取前端并发清单和后端用户、部门、角色接口文档。

## 2. 模块目标

- 实现用户、部门、角色页面闭环。

## 3. 模块边界

- 只做基础资料页。
- 不实现登录、人脸、打卡、异常页。

## 4. 前置依赖

- `FE-01`
- 后端依赖 `BE-01`

## 5. 固定接口清单

1. `GET /api/user/list`
2. `POST /api/user/add`
3. `PUT /api/user/update`
4. `DELETE /api/user/{id}`
5. `GET /api/department/list`
6. `POST /api/department/add`
7. `PUT /api/department/update`
8. `DELETE /api/department/{id}`
9. `GET /api/role/list`
10. `POST /api/role/add`
11. `PUT /api/role/update`
12. `DELETE /api/role/{id}`

## 6. 固定字段与状态约定

- 用户表单必须可选择部门和角色。
- 列表支持分页和关键字筛选。
- `PUT /api/user/update` 中 `password` 为可选字段；未传或空白时保留原密码。
- 字段以 API 文档为准。

## 7. 代码目录范围

- `src/views/user/`
- `src/views/department/`
- `src/views/system/role/`
- `src/api/`
- `src/components/`

## 8. 详细开发步骤

1. 封装用户、部门、角色 API。
2. 实现三个列表页。
3. 实现分页、查询和空态。
4. 实现新增、编辑、删除交互。
5. 打通用户表单与部门、角色联动。
6. 补齐错误提示和构建验证。

## 9. 最小验证命令

```bash
npm run build
```

## 10. 完成标准

- 三类页面形成闭环。
- 表单联动正常。
- 可直接联调 `BE-01`。

## 11. 禁止改动项

- 不在本模块处理设备、人脸、打卡和异常页面。

## 12. 文档同步要求

- 前端字段不一致时，先回看后端 API 文档，不自行发明字段名。
