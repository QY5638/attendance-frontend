# FE-02 概览与统计模块开发指南

## 1. 文档说明

- 本文件用于指导 `FE-02` 的独立开发。
- 开发前先读取前端并发清单和后端统计接口文档。

## 2. 模块目标

- 实现 `/dashboard` 和 `/statistics` 页面，并体现 AI 统计摘要与风险画像能力。

## 3. 模块边界

- 只做概览、趋势、智能摘要、风险画像、报表和导出入口。
- 不做复核和系统配置页。

## 4. 前置依赖

- `FE-01`
- 后端依赖 `BE-08`，预警摘要依赖 `BE-06`

## 5. 固定接口清单

1. `GET /api/statistics/personal`
2. `GET /api/statistics/department`
3. `GET /api/statistics/exception-trend`
4. `GET /api/statistics/summary`
5. `GET /api/statistics/department-risk-brief`
6. `GET /api/statistics/export`
7. `GET /api/warning/list`

## 6. 固定字段与状态约定

- `GET /api/statistics/personal` 按当前登录会话推导统计用户，前端不传 `userId`。
- `GET /api/statistics/department` 仅 `ADMIN` 可访问，默认统计全局部门汇总，前端不传 `deptId`，返回中的 `deptName` 固定为 `全部部门`。
- `GET /api/statistics/summary` 存在角色分流：`ADMIN` 查看全局摘要，`EMPLOYEE` 查看本人个人摘要。
- `GET /api/statistics/exception-trend` 查询参数和图表字段以 `startDate`、`endDate`、`periodType`、`points[]` 为准。
- `GET /api/statistics/department-risk-brief` 返回 `deptId`、`deptName`、`riskScore`、`riskSummary`、`manageSuggestion`。
- `GET /api/warning/list` 是分页列表接口；首页预警摘要需基于列表结果提炼，不要按独立摘要接口接入。
- `GET /api/statistics/export` 返回 `text/csv;charset=UTF-8` 文件下载响应，本轮不默认放开员工导出权限。
- 首页展示统计摘要、预警摘要和 AI 摘要。
- 统计页支持筛选、趋势图、风险画像、智能建议和导出。

## 7. 代码目录范围

- `src/views/dashboard/`
- `src/views/statistics/`
- `src/api/`
- `src/components/`

## 8. 详细开发步骤

1. 搭建 dashboard 概览卡片区域。
2. 接入预警列表分页结果、核心统计接口和统计摘要接口。
3. 搭建 statistics 页面筛选区、图表区和风险画像区。
4. 接入趋势接口、风险画像接口和导出按钮。
5. 展示智能管理建议和摘要卡片。
6. 补齐空态、加载态和错误提示。

## 9. 最小验证命令

```bash
npm run build
```

## 10. 完成标准

- dashboard 可展示摘要。
- statistics 可展示统计、趋势和风险画像。
- 页面可展示 AI 摘要与管理建议。
- 导出入口可用。

## 11. 禁止改动项

- 不在本模块处理系统配置和复核流程。

## 12. 文档同步要求

- 图表字段和导出参数以 API 文档为准，发现差异时先确认契约。
