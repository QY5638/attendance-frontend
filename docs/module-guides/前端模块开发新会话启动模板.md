# 前端模块开发新会话启动模板

## 1. 适用场景

- 用于在新会话中启动某个前端模块的独立开发。
- 适用于 `D:\Graduation project\frontend` 仓库。
- 目标是让新会话在最少来回确认的情况下，快速进入前端模块开发或联调。

## 2. 前端开发前最少需要提供的文件

以下文件建议在新会话一开始就明确要求读取：

1. `D:\Graduation project\frontend\AGENTS.md`
2. `D:\Graduation project\backend\docs\progress\前端并发开发模块清单.md`
3. `D:\Graduation project\backend\docs\api\API接口设计文档.md`
4. `D:\Graduation project\backend\docs\architecture\模块划分说明书.md`

## 3. 建议额外补充的文件

如果希望前端实现时减少来回沟通，建议再补充：

1. `D:\Graduation project\backend\docs\progress\前后端模块联调依赖清单.md`
2. `D:\Graduation project\backend\docs\progress\后端并发开发模块清单.md`
3. 对应前端模块开发指南，例如：`D:\Graduation project\frontend\docs\module-guides\FE-04-人脸采集模块开发指南.md`
4. 如字段较复杂，再补：
   - `D:\Graduation project\backend\docs\database\数据库设计文档.md`
   - `D:\Graduation project\backend\docs\database\数据库E-R关系说明.md`

## 4. 前端能否与后端并发开发

- 可以并发开发。
- 前端可以先基于 API 契约完成页面结构、交互流程、状态管理和假数据联调。
- 但真实联调必须以对应后端模块已经实现为前提。
- 除 `FE-01` 外，其余前端模块默认依赖 `FE-01` 提供登录态、角色菜单和受保护路由。

## 5. 使用原则

1. 每次只认领一个 `FE-xx` 模块。
2. 先按接口契约开发，不要跨模块顺手改别的页面组。
3. 如果接口未实现，先完成页面骨架和交互闭环。
4. 如果发现接口字段与文档不一致，优先回看 API 文档并记录差异。
5. 完成后优先做与本模块直接相关的最小验证，例如页面加载、构建检查、接口联调。

## 6. 通用新会话启动模板

把下面这段直接发到新会话即可：

```text
仓库路径：D:\Graduation project\frontend

我要开始前端模块开发。

请先读取这些文件建立上下文：
- D:\Graduation project\frontend\AGENTS.md
- D:\Graduation project\backend\docs\progress\前端并发开发模块清单.md
- D:\Graduation project\backend\docs\progress\前后端模块联调依赖清单.md
- D:\Graduation project\backend\docs\api\API接口设计文档.md
- D:\Graduation project\backend\docs\architecture\模块划分说明书.md
- D:\Graduation project\frontend\docs\module-guides\<替换成对应 FE 模块开发指南>.md

当前任务：开发 `FE-xx` 模块。

本次约束：
- 只开发 `FE-xx`，不要跨模块顺手改其他页面
- 默认中文回答，称呼我“主人”
- 先阅读上下文，再给出设计
- 我确认设计后，再写实现计划
- 我确认后，再开始编码
- 优先最小必要改动，沿用现有项目风格
- 技术栈固定为 Vue3 + Element Plus
- 如后端接口尚未实现，可先按契约完成页面骨架与交互
- 如需真实联调，先核对对应后端模块是否已经完成
- 完成后执行与本模块直接相关的最小验证
- 不要主动 commit、push、创建 PR，除非我明确要求

如果需要，请为本次开发使用独立分支或 worktree。
现在开始。
```

```text
- 默认中文回答，称呼我“主人”

- 我确认设计后，再写实现计划
- 我确认后，再开始编码

```
