# FE-01 公共壳层与认证模块开发指南

## 1. 文档说明

- 本文件用于指导 `FE-01` 的独立开发。
- 开发前先读取：
  - `D:\Graduation project\frontend\AGENTS.md`
  - `D:\Graduation project\backend\docs\progress\前端并发开发模块清单.md`
  - `D:\Graduation project\backend\docs\api\API接口设计文档.md`

## 2. 模块目标

- 完成前端基础壳层、登录页、菜单和路由守卫。
- 为后续所有页面模块提供统一入口。

## 3. 模块边界

- 只做认证和后台主壳层。
- 不实现具体业务页。

## 4. 前置依赖

- 后端依赖 `BE-01`。
- 当前前端仓库已有 `src/` 目录骨架，但前端可运行工程能力需在本模块补齐。

## 5. 固定接口清单

1. `POST /api/auth/login`

## 6. 固定字段与状态约定

- 登录成功返回使用 `token`、`roleCode`。
- 未登录访问后台必须跳回 `/login`。
- 菜单显示需与 `roleCode` 匹配。

## 7. 代码目录范围

- `src/views/login/`
- `src/layout/`
- `src/router/`
- `src/store/`
- `src/api/`
- `src/utils/`

## 8. 详细开发步骤

1. 补齐前端工程基础脚本和依赖清单。
2. 搭建后台主布局。
3. 实现登录页和表单校验。
4. 接入登录 API。
5. 实现 token 存储和退出登录。
6. 实现路由守卫。
7. 基于 `roleCode` 控制菜单显示。
8. 为后续页面模块预留路由挂载点。

## 9. 最小验证命令

```bash
npm run build
```

## 10. 完成标准

- 登录后可进入后台。
- 未登录访问后台被拦截。
- 菜单能按角色显示。
- 后续模块可以在壳层内挂载。

## 11. 禁止改动项

- 不在本模块实现用户、打卡、异常等具体业务页。

## 12. 文档同步要求

- 若登录字段映射变化，先以后端 API 文档为准，再同步前端实现。
