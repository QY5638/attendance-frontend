# FE-04 人脸采集模块开发指南

## 1. 文档说明

- 本文件用于指导 `FE-04` 的独立开发。
- 开发前先读取前端并发清单和后端人脸接口文档。

## 2. 模块目标

- 完成人脸录入与验证页面。

## 3. 模块边界

- 只做采集、上传、录入和验证反馈。
- 不做打卡提交流程。

## 4. 前置依赖

- `FE-01`
- 后端依赖 `BE-03`

## 5. 固定接口清单

1. `POST /api/face/register`（当前登录用户自助录入）
2. `POST /api/face/verify`（当前登录用户自助验证）

## 6. 固定字段与状态约定

- `POST /api/face/register` 与 `POST /api/face/verify` 请求体只传 `imageData`；若携带 `userId`，后端会忽略并以当前会话用户覆盖。
- `POST /api/face/register` 与 `POST /api/face/verify` 响应中的 `userId` 表示当前会话解析出的用户编号。
- `POST /api/face/verify` 中 `registered=false` 表示未录入人脸，`matched=false` 表示验证失败。
- 页面展示录入结果、验证结果和必要分数反馈。
- 不显示敏感原始特征数据。

## 7. 代码目录范围

- `src/views/face/`
- `src/api/`
- `src/components/`

## 8. 详细开发步骤

1. 搭建录入区域和上传入口。
2. 接入录入接口。
3. 接入验证接口。
4. 展示成功、失败、重试反馈。
5. 处理未录入和验证失败场景。

## 9. 最小验证命令

```bash
npm run build
```

## 10. 完成标准

- 人脸录入页面可用。
- 验证结果反馈清晰。

## 11. 禁止改动项

- 不在本模块实现考勤记录页和打卡提交流程。

## 12. 文档同步要求

- 页面字段和返回结果以 API 文档为准。
